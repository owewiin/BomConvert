SchBom Convert (V1.4版本)
一個用於轉換和格式化 BOM (Bill of Materials) 清單的 WPF 應用程式。

## 功能特色
📋 BOM 資料處理

識別零件類型：識別 SMD、DIP 等不同封裝的電子零件，分類順序可自訂，特殊零件（如合金電阻）不會被一般類型覆蓋
分類排序：按照零件類型、規格、封裝尺寸自動分類和排序
替代料處理：支援替代料資訊的解析和格式化
NC 零件支援：特別處理 NC (No Connect) 零件，包含視覺區別
轉換資訊顯示：即時查看處理過程中的所有訊息和警告

📤 匯出功能

標準化 Excel 輸出：生成符合公司規範的 BOM 清單
多分類料表：自動分為 SMD 料表、DIP 料表、其他料表
產品資訊完整：包含產品編號和產品名稱欄位
視覺改善設計：SMD 尺寸採用白色字體，提升專業外觀
自動計算：自動計算小計和總計
工程人員資訊：包含研發助理、Layout 人員、線路設計人員資訊

⚙️ 個人化設定

人員設定管理：可自訂工程人員名單，支援新增/刪除操作
設定檔案儲存：設定自動儲存到 JSON 檔案，支援 NAS 環境
自動開啟檔案：可設定匯出後自動開啟生成的檔案
轉換資訊面板：左側欄新增按鈕，查看所有處理訊息
欄位寬度自訂：可調整 Excel 輸出的欄位寬度設定

系統需求

Windows 10 或更新版本
.NET 8.0 或更新版本
Microsoft Excel (用於開啟生成的檔案)

使用說明
1. 載入 BOM 檔案

點擊「選擇檔案」按鈕
選擇要處理的 Excel 檔案 (.xlsx, .xls, .xlt)
系統會自動載入並解析資料

2. 預覽資料

點擊「📝 預覽」按鈕
檢查解析結果和分類是否正確
確認替代料格式：應為「規格(廠商/代理商)」格式

3. 設定工程人員

點擊「⚙️ 人員設定」
設定或選擇：


研發助理人員
Layout 人員
線路設計人員


新增人員：在對應欄位輸入名稱，點擊綠色「+」按鈕
刪除人員：選擇要刪除的人員，點擊紅色「-」按鈕
點擊「💾 儲存設定」

4. 匯出 BOM 清單

點擊「📤 匯出 Excel」
選擇儲存位置和檔案名稱
系統會生成標準化的 BOM 清單，包含產品名稱欄位和視覺改善

5. 查看轉換資訊

點擊「ℹ️ 轉換資訊」按鈕
查看所有處理過程中的訊息、警告和錯誤

檔案格式要求
輸入 Excel 格式
預期的 Excel 檔案結構（從第 6 行開始）：
欄位說明A項次B零件名稱C規格D封裝E數量F零件編號G單價H廠商I備註J製程K替代料1L中國零件編號M中國廠商N中國替代料O替代料2
輸出格式
生成的 Excel 檔案包含：

公司標頭資訊
產品編號和產品名稱欄位
按類型分組的零件清單 (SMD/DIP/其他)
SMD 尺寸白色字體視覺改善
自動計算的小計和總計
工程人員資訊和日期
NC 零件特殊標記和視覺區別

零件識別規則
SMD 零件分類
系統會自動識別以下 SMD 零件類型：

電阻、合金電阻
電容、鉭電
電感、三端濾波器
二極體、蕭特基、稽納、LED
電晶體、MOSFET
各種 IC (MCU, OP, DC/DC, LDO 等)
保護元件 (保險絲, BEAD, TVS, ESD)
Connector 和 Sensor

DIP 零件分類

DIP 排針
DIP 排母
DIP 端子座（只包含「端子座」關鍵字，不再包含「端子」）
DIP 歐規（支援「歐規」、「歐規端子」、「歐規母座」、「歐規公座」）
電解電容、安規電容
電感、CHOKE、變壓器
二極體、橋式整流器
主動元件 (BJT, MOSFET, IGBT, RELAY)
模組和感測器
人機介面元件(DSPL)
機構件 (散熱片、銅塊等)

排序邏輯

SMD 零件：按尺寸 (0603→0805→1206...) 和類型排序
DIP 零件：按元件功能和重要性排序
電阻：按數值大小排序 (按照純數字大小排序 EX:1R→1K→47K→220R....)
電容：按容值大小排序

SMD 尺寸視覺處理

包裝欄位中的 SMD 尺寸 (0402, 0603, 0805, 1206, 1210, 2010, 1812, 2512, 2728, 3920, SR3920) 自動設為白色字體
適用於一般項目和 NC 項目("預留")
提升 BOM 清單的專業外觀

產品資訊完整性

每個料表標題下方自動新增"產品名稱:"欄位
提供更完整的產品識別資訊
符合標準 BOM 格式要求

已知問題與修復
V1.21 修復內容

問題：刪除工程人員時程式閃退
原因：ComboBox 綁定衝突和缺少異常處理
解決方案：改用 ObservableCollection 並添加完整異常處理
問題：新增人員後出現 ItemsControl 不一致錯誤
原因：使用 List 而非 ObservableCollection
解決方案：統一使用 ObservableCollection 進行資料綁定
